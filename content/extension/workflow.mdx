# Explore vs Tests

Glubean uses two directories because local exploration and committed verification are different activities.

## The layout

When you run `npx @glubean/cli init`, you get:

```
my-project/
├── explore/   # scratchpad checks
├── tests/     # committed verification suite
└── data/      # shared JSON/CSV for data-driven checks
```

The extension reads this folder structure and groups your tests in the **Test Explorer** sidebar accordingly.

## `explore/`

Use `explore/` when you are:

- trying an endpoint for the first time
- reproducing a bug
- asking AI to generate a first draft
- iterating quickly without worrying about CI

These files are meant to be cheap to rewrite.

Many teams choose to gitignore `explore/` entirely.

## `tests/`

Move a file to `tests/` when it is stable enough to be part of the regression suite.

That is usually the only change required.

The same file you ran locally can now run in CI with `glubean run ./tests`.

## Rule of thumb

- `explore/` is for speed
- `tests/` is for trust
