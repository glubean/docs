# Traces & Diff

Every request sent through Glubean becomes a trace. This is the fastest way to understand what actually happened.

## What opens after a run

After a gutter or Test Explorer run, the extension can open:

- a trace viewer for request and response details
- a result viewer for pass or fail summary

If your test makes multiple requests, the trace keeps them together in execution order.

### Copy as cURL

While viewing a trace, you can copy a captured request as a ready-to-run shell command.

1. Open the Command Palette (`Cmd+Shift+P` / `Ctrl+Shift+P`)
2. Search for **Glubean: Copy as cURL**
3. Paste it into your terminal.

## Trace history

Glubean keeps recent traces per test. Above each test you can open the newest saved trace from the `Trace (N)` CodeLens.

You can step backward and forward through history:
- **Older trace:** `Alt + [` (Mac/Win)
- **Newer trace:** `Alt + ]` (Mac/Win)

## Diff with previous run

Run **Glubean: Diff with Previous Run** from the Command Palette while a trace is open. VS Code opens the native side-by-side diff view so you can see payload changes immediately.

> Raw trace artifacts are still written to `.glubean/traces/*.trace.jsonc` for archival and CLI workflows.
