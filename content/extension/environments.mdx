# Environments & Secrets

This page focuses on the VS Code extension workflow (status bar switching, hover preview, and run behavior). For the canonical rules (file model, resolution order, system fallback, and `${...}` mapping), see [Environments & Secrets](/guides/environments).

## Environment files

Use the standard Glubean env file model (`.env`, `.env.staging`, `.env.prod` and matching `.secrets` files) from the canonical guide: [Environments & Secrets](/guides/environments).

---

## Switching environments

The active environment is shown in the status bar at the bottom of the window. Click it to open a quick-pick menu:

```
⚙ .env  ▾
```

Selecting `.env.staging` immediately switches all subsequent runs to load variables from that file. The matching `.env.staging.secrets` is loaded automatically — no extra step.

You can also switch from the Command Palette:

```
Glubean: Select Environment
```

---

## Accessing variables in tests

Use `ctx.vars` for regular variables and `ctx.secrets` for sensitive values:

```typescript
export const getUser = test("get-user", async (ctx) => {
  const baseUrl = ctx.vars.require("BASE_URL");
  const token = ctx.secrets.require("API_TOKEN");

  const res = await ctx.http.get(`${baseUrl}/users/me`, {
    headers: { Authorization: `Bearer ${token}` },
  });
});
```
For full API semantics (`get` vs `require`, validation behavior), see [SDK Environment & Secrets](/sdk/env-and-secrets).

---

## Hover preview

Hover over any `vars.require("KEY")` or `secrets.require("KEY")` call in your editor to see the resolved value from the currently selected environment:

```typescript
const baseUrl = ctx.vars.require("BASE_URL");
//                                  ^
//   hover → BASE_URL = "https://api.staging.example.com"
```

Secret values are masked in the hover tooltip.
