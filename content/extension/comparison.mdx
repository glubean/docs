# Comparison

## Different tools, different philosophies

Glubean, Postman, and REST Client all deal with APIs — but they solve different problems.

**Postman** is a standalone GUI application. You fill in fields, click buttons, organise requests into collections, and share them via cloud sync. Great for teams with non-developers.

**REST Client** (the VS Code extension) lets you write `.http` files with raw HTTP syntax. Simple and lightweight — great for one-off requests.

**Glubean** is a developer-owned API verification platform. Your verification code is TypeScript, versioned in git, run from the editor or CI. Beyond HTTP, plugins support browser automation, GraphQL, gRPC, and anything Node.js supports. `explore/` replaces Postman collections (with `test.pick` as the equivalent of Postman Examples), `tests/` is your CI regression suite. Vars and secrets are separated at the SDK level — not by convention, by architecture. And it's AI-native: MCP server + skill + schema inference enable an AI self-healing loop where tests are generated, executed, and fixed without leaving the chat.

None of these is objectively better. The right choice depends on how your team works.

---

## Who each tool is best for

**Glubean** fits naturally if you:
- Want your API collection in git, not in a proprietary cloud
- Need real assertions, not just "200 OK"
- Work with multi-step flows (create → fetch → verify → cleanup)
- Use AI to generate and maintain tests (MCP + skill closed loop)
- Need the same verification to run locally, in CI, and against production
- Want to verify more than HTTP — browser, GraphQL, gRPC, crypto, or any Node.js code

**Postman** fits naturally if you:
- Prefer a visual interface over writing code
- Work with non-developer teammates (QA, support, product) who need to send requests without a coding setup
- Want a standalone tool independent of your editor

**REST Client** fits naturally if you:
- Want the simplest possible thing — just type a request and send it
- Don't need assertions or scripting, just raw request/response visibility
- Already have `.http` files checked into your repository

## Feature comparison

| | Glubean | Postman | REST Client |
|---|---|---|---|
| Where it lives | VS Code / Cursor | Standalone app | VS Code |
| Request authoring | TypeScript code | GUI form | `.http` file syntax |
| Beyond HTTP | Plugins: browser, GraphQL, gRPC, any Node.js lib | Limited | No |
| Version control | Git-native — `.ts` files, PR review, merge | Export/import JSON | Git-native — `.http` files |
| Assertions | Fluent `expect`, schema validation, custom metrics | Test scripts (JS) | None |
| Data-driven | `test.each` (all cases) + `test.pick` (= Postman Examples) | Collection Runner | Not supported |
| Multi-step flows | `.step()` builder with shared state | Collections + variables | Not supported |
| Result history & diff | Per-test history, prev/next navigation, VS Code diff | Not built-in | Not built-in |
| Vars & secrets | Separated by architecture (`.env` + `.env.secrets`) | Mixed in one environment | Variables in `.env` |
| AI integration | AI-native: MCP server, skill, schema inference, self-healing loop | Limited | Limited |
| CI integration | `glubean run` in any pipeline | Newman CLI | Not supported |
| Debugging | Full VS Code debugger — breakpoints in any Node.js code | Console-only | Not supported |
| Pricing | Free, open source | Free + paid tiers | Free, open source |

---

## Summary

REST Client is the right tool when you want a quick, no-setup way to fire off requests from inside VS Code. It's a great scratchpad.

Postman is the right tool when your team needs a shared, visual workspace — especially if non-developers are involved.

Glubean is the right tool when API verification is a first-class part of your codebase: git-safe, developer-owned, AI-accelerated, and running the same code from your editor to production.
