# Common Recipes

These are the command patterns most teams keep using.

## CI run with machine-readable output

```bash
glubean run ./tests \
  --result-json .glubean/last-run.json \
  --reporter junit:test-results.xml
```

Use this when CI needs both a Glubean result file and a JUnit report.

## Switch environments without changing code

```bash
glubean run
glubean run --env-file .env.staging
glubean run --env-file .env.prod
```

The CLI automatically pairs each env file with its matching `.secrets` file. The canonical rules live in [Environments & Secrets](/cli/environments).

## Reuse shell values without copying secrets

```bash
export GLUBEAN_URL=https://api.example.com
export GLUBEAN_API_KEY=sk_live_xxx

printf "BASE_URL=\${GLUBEAN_URL}\n" > .env
printf "API_KEY=\${GLUBEAN_API_KEY}\n" > .env.secrets

glubean run
```

This keeps test-facing names stable while reusing shell-level values.

## Run only one test or one named example

```bash
glubean run ./tests/users.test.ts --filter "create-user"
glubean run ./tests/users.test.ts --pick "edge,invalid"
```

## Run by tags

```bash
glubean run --tag smoke --tag critical
glubean run --tag auth --tag smoke --tag-mode and
```

## Save more detail while debugging

```bash
glubean run ./tests/flaky.test.ts --verbose --emit-full-trace
```
