# CLI & Config

Use the CLI when you want the stable, scriptable surface of Glubean.

The extension is the best local UI, but the CLI is the source of truth for:

- CI execution
- result files and JUnit output
- environment switching
- uploads to Cloud
- repeatable team workflows

## The commands that matter most

- `npx @glubean/cli init` to scaffold a project
- `glubean run` to execute checks
- `glubean scan` to export metadata
- `glubean validate-metadata` to keep generated metadata honest in CI
