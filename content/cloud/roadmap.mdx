# Roadmap

What's available in Glubean Cloud today and what's coming next.

## Available Now

These features are live and ready to use.

### Core

- **Run upload & storage** — `glubean run --upload` sends results to Cloud
- **Run history** — browse, search, and inspect past runs
- **Real-time streaming** — watch test results appear live during a run
- **Artifact viewer** — inspect traces, screenshots, and response snapshots in the browser
- **Multiple auth methods** — project tokens, env vars, credentials file, deno.json config

### Collaboration

- **Team projects** — invite members with role-based access (owner, admin, member, viewer)
- **Public projects** — shareable links, no login required
- **Status badges** — embeddable pass/fail and pass-rate badges

### Integrations

- **GitHub Actions** — upload from CI with a single env var
- **Webhooks** — `run.completed` and `run.failed` event notifications
- **GitHub / Google OAuth** — sign in with existing accounts

### Developer Experience

- **CLI upload** — one flag (`--upload`) to send results
- **Token scoping** — project tokens with granular permissions (`runs:read`, `runs:write`, etc.)
- **Environment management** — env groups for project-level variables

---

## Coming Soon

These features are designed and partially built. They will be enabled in upcoming releases.

### Analytics

- **Pass rate trend** — 30-day rolling pass rate chart per project
- **Flaky test detection** — automatically identify tests with inconsistent results, ranked by flaky score
- **Test duration trend** — track P50/P95/P99 latency per test over time
- **Failure distribution** — aggregate failures by class and pattern
- **Regression detection** — flag significant performance changes with commit correlation

### Monitoring

- **Prometheus metrics export** — expose project metrics in Prometheus format for external dashboards
- **Email alerts** — notifications on run failure (Pro plan and above)
- **Slack / Discord alerts** — channel notifications on run events (Team plan)

### CI Integration

- **GitHub commit status checks** — automatic pass/fail status on pull requests
- **Auto-build on push** — trigger Cloud runs from git push events

### Execution

- **Scheduled runs** — cron and interval-based test execution in Cloud
- **Cloud workers** — managed execution environments (no local CLI needed)
- **Self-hosted workers** — run on your own infrastructure with Cloud reporting
