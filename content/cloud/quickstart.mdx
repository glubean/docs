# Upload Your First Run

This guide takes you from a working local test to seeing results in Glubean Cloud. Takes about 5 minutes.

## Prerequisites

- Glubean CLI installed ([Installation](/getting-started/installation))
- A test that runs locally (`glubean run` works)

## 1. Create a Cloud account

Go to [app.glubean.com](https://app.glubean.com) and sign in with GitHub or Google.

A default project is created automatically when you sign up.

## 2. Get a project token

1. Open your project in the Cloud dashboard.
2. Go to **Settings → Tokens**.
3. Click **Create Token** and copy the value (starts with `gpt_`).

## 3. Configure the token

Pick the method that fits your setup. For a full explanation of token types, see [Tokens & Auth](/cloud/tokens).

### Option A: Environment file (recommended for local dev)

Uses a **project token** (`gpt_`). Add to `.env.secrets` in your project root:

```bash
GLUBEAN_TOKEN=gpt_your_project_token
GLUBEAN_PROJECT_ID=prj_abc123
```

> `.env.secrets` is gitignored by default. Never commit tokens to version control.

The CLI auto-loads `.env` + `.env.secrets` by default. For per-environment tokens, use `--env-file`:

```bash
# .env.staging.secrets
GLUBEAN_TOKEN=gpt_staging_token

# .env.production.secrets
GLUBEAN_TOKEN=gpt_production_token
```

```bash
glubean run --env-file .env.staging --upload
```

The CLI loads both `.env.staging` and `.env.staging.secrets` automatically.

### Option B: Environment variable (recommended for CI)

Uses a **project token** (`gpt_`). Store it in your CI provider's secrets:

```yaml
# GitHub Actions example
env:
  GLUBEAN_TOKEN: ${{ secrets.GLUBEAN_TOKEN }}
  GLUBEAN_PROJECT_ID: ${{ vars.GLUBEAN_PROJECT_ID }}
```

### Option C: deno.json (quick start / public demos)

Uses a **project token** (`gpt_`). Inline in your project config:

```json
{
  "glubean": {
    "cloud": {
      "token": "gpt_your_project_token",
      "projectId": "prj_abc123"
    }
  }
}
```

> Only use this for tokens with limited scope. The CLI warns if your deno.json is tracked by git.

### Option D: Global login (personal use across projects)

Uses a **user API key** (`gb_`) or project token. Stores credentials globally:

```bash
glubean login
```

Saves to `~/.glubean/credentials.json`. A user API key gives access to all your projects from one credential. For project-specific tokens, prefer Option A or B.

## 4. Upload a run

```bash
glubean run --upload
```

By default, `glubean run` executes all `*.test.ts` files in your `tests/` directory. Add `--upload` to send results to Cloud after execution.

### Running specific tests

```bash
# Run a single file
glubean run ./tests/auth.test.ts --upload

# Run a specific directory
glubean run ./tests/api/ --upload

# Filter by test name/id (substring match)
glubean run --filter "login" --upload

# Filter by tag (exact match, OR logic)
glubean run --tag "smoke" --upload

# Multiple tags with AND logic
glubean run --tag "smoke" --tag "auth" --tag-mode and --upload

# Run explore tests (not committed regression tests)
glubean run ./explore --upload
```

For all execution options, see [CLI Reference](/cli/reference).

### Example output

```text
✓ health        12ms
✓ list-users    45ms

2 passed, 0 failed

Uploaded → https://app.glubean.com/p/my-project/runs/abc123
```

## 5. View in Cloud

Click the URL from the output, or open the Cloud dashboard and navigate to your project. You'll see:

- **Run summary** — pass/fail counts, duration, timestamp
- **Test results** — each test with status and timing
- **Events** — assertions, logs, and traces expandable per test

## CI integration

Add `--upload` to your existing CI command:

```bash
glubean run ./tests --upload
```

Every CI run uploads to Cloud automatically. The team sees results without checking CI logs.

```yaml
# Full GitHub Actions example
name: api-tests
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: denoland/setup-deno@v2
        with:
          deno-version: v2.x
      - run: deno install -Agf -n glubean jsr:@glubean/cli
      - run: glubean run ./tests --upload
        env:
          GLUBEAN_TOKEN: ${{ secrets.GLUBEAN_TOKEN }}
```

## Token resolution order

The CLI looks for credentials in this order (first match wins):

1. `--token` CLI flag
2. `GLUBEAN_TOKEN` environment variable
3. `.env` / `.env.secrets` file in project root
4. `deno.json` → `glubean.cloud.token`
5. `~/.glubean/credentials.json` (from `glubean login`)
