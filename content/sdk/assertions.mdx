# Assertions

Glubean gives you four verification tools with different failure behavior.

## `ctx.expect`

Use `expect` for normal assertions. It is soft by default, so several failures can be collected in one run.

```typescript
ctx.expect(body.role).toBe("admin");
ctx.expect(body.features).toContain("beta-access");
```

If a particular expectation should stop execution immediately, chain `.orFail()`:

```typescript
ctx.expect(res.status).toBe(200).orFail();
```

## `ctx.assert`

Use `assert` when a condition is so fundamental that the test should fail right there.

```typescript
ctx.assert(res.ok, "Request should succeed");
```

## `ctx.validate`

Use this when manual property checks would be noisy and a schema is clearer.

It works with libraries such as Zod, Valibot, ArkType, and other parse-style validators.

## `ctx.warn`

Use warnings for budgets and best-practice checks that should be visible without failing the run.

```typescript
ctx.warn(durationMs < 500, "Response should stay under 500ms");
```
