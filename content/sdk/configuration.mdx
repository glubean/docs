# Configuration

Use `configure()` when several checks in one file need the same runtime setup.

This keeps repeated environment access, headers, and plugin wiring out of each individual test.

## File-level setup

Use `configure()` to set up shared HTTP defaults, variables, and secrets once per file.

```typescript
import { configure, test } from "@glubean/sdk";

export const { http, vars, secrets } = configure({
  vars: {
    baseUrl: "{{BASE_URL}}",
  },
  secrets: {
    apiKey: "{{API_KEY}}",
  },
  http: {
    prefixUrl: "{{BASE_URL}}",
    headers: { Authorization: "Bearer {{API_KEY}}" },
  },
});

export const listUsers = test("list-users", async (ctx) => {
  const data = await http.get("users?limit=5").json<{ users: any[] }>();
  ctx.expect(data.users.length).toBe(5);
  ctx.log(`Tested against ${vars.baseUrl}`);
});
```

Values declared in `configure()` are resolved lazily at runtime, which keeps metadata scans safe and prevents top-level env reads from crashing tooling.

## Plugin registration

`configure()` is also where you register plugins, such as the GraphQL client or custom API wrappers.

```typescript
import { configure, test } from "@glubean/sdk";
import { graphql } from "@glubean/graphql";

const { gql } = configure({
  plugins: {
    gql: graphql({
      endpoint: "{{GRAPHQL_URL}}",
      headers: { Authorization: "Bearer {{API_KEY}}" },
      throwOnGraphQLErrors: true,
    }),
  },
});

export const checkGraphql = test("graphql", async (ctx) => {
  const { data } = await gql.query(`{ me { id } }`);
});
```
