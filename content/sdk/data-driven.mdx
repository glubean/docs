# Data-Driven

Use data-driven tests when the logic stays the same but the input examples change.

## Pick the right shape

| Use case | Best fit |
| --- | --- |
| Run the same test for every row | `test.each()` |
| Run one named example at a time | `test.pick()` |
| Load rows from files | `fromCsv`, `fromYaml`, `fromDir`, `fromDir.concat` |
| Load named examples from files | `fromDir.merge()` |

## `test.each()`

Use `test.each()` when every row should run:

```typescript
export const getUser = test.each([
  { id: 1, expected: 200 },
  { id: 999, expected: 404 },
])("get-user-$id", async (ctx, row) => {
  const res = await ctx.http.get(`${ctx.vars.require("BASE_URL")}/users/${row.id}`);
  ctx.expect(res.status).toBe(row.expected);
});
```

## `test.pick()`

Use `test.pick()` when you want named examples and selective reruns:

```typescript
export const createUser = test.pick({
  normal: { name: "Alice", expected: 201 },
  invalid: { name: "", expected: 400 },
})("create-user-$_pick", async (ctx, example) => {
  const res = await ctx.http.post("/users", { json: { name: example.name } });
  ctx.expect(res.status).toBe(example.expected);
});
```

From the CLI:

```bash
glubean run tests/users.test.ts --pick invalid
```

## File helpers

- `fromCsv(path)` loads rows from CSV
- `fromYaml(path)` loads rows or named examples from YAML
- `fromDir(path)` treats each file as one row
- `fromDir.concat(path)` concatenates arrays from many files
- `fromDir.merge(path)` merges named example maps from many files

`fromDir.merge()` pairs especially well with [git-safe personal test data](/sdk/local-data).

## Rule of thumb

- use `test.each()` for CI coverage
- use `test.pick()` for local exploration and deterministic named examples
